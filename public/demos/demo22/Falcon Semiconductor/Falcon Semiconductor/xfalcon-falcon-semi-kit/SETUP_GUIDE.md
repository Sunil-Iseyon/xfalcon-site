# Falcon Semiconductor — Setup Guide

Step-by-step for using this analytics kit.

## Quick Start (5 steps)

1. **Open** `index.html` in a browser — this is the portal landing page.
2. **Check** the KPI cards reflect expected FY25 values (Revenue $3.71B, GM% 50.3%, B:B 1.08).
3. **Click** into any numbered dashboard from the portal directory (1–11).
4. **Use** the filter bar at the top of each dashboard (Year, End Market, Channel, Compare Years toggle).
5. **Return** to the portal via "← Back to Portal" in the top-right of any dashboard.

## Dashboard build checklist

Before shipping any dashboard, verify:

- [ ] Topbar matches Tech Innovation theme (dark `#1E1E1E` band, xF logo, right-side back link)
- [ ] ≥6 KPI cards, single-year, explicit year label in each
- [ ] ≥4 chart visualizations
- [ ] ≥1 detail table with YoY column and ≥7 rows
- [ ] ≥1 disclaimer/caveat line about filters applied
- [ ] Compare Years toggle present and working
- [ ] FY25 revenue reproduces as $3.71B under default filters (see VALIDATION_BENCHMARKS.md)
- [ ] Chart.js v4 CDN exact URL loaded
- [ ] `type: 'bar'` + `indexAxis: 'y'` for horizontal bars — never `type: 'barh'`
- [ ] All chart colors are hex strings, never `var(--...)`
- [ ] Back link `href="index.html"` (never `#` or `history.back()`)

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| All charts blank | Chart.js CDN URL wrong or JS syntax error | Check CDN URL exactly matches `chart.umd.js`; run `node --check` on `<script>` content |
| Revenue wrong by 2× | Computed B:B from order/shipment ratio | Use `FACT_BACKLOG.BOOK_TO_BILL_RATIO` instead |
| End-market share empty | Joined FACT_SHIPMENTS.END_MARKET_KEY directly | Route via `DIM_CUSTOMER.END_MARKET_CODE` (shipments has no EM key) |
| FY25 query returns 0 rows | Trailing space in `FISCAL_YEAR` column | Use `TRIM(FISCAL_YEAR) = 'FY25'` or `LIKE 'FY25%'` |
| Bookings much higher than revenue | Sample orders included | Add `WHERE ORDER_TYPE <> 'Sample'` |
| Filter dropdown has duplicates | HTML + JS both populating | Clear `innerHTML` before JS populates |

## Running queries

This kit assumes IDA MCP connector `mcp__0f5a7fbd-d3a0-4d09-80d5-e325ec2e51bb__` (the new Falcon Semi database) — not the previous consumer-goods connector. If other IDA connectors appear:
1. List tables on each with `ida_get_knowledge(knowledge_type='tables', detail_level='names')`
2. Look for DIM_CUSTOMER, FACT_SHIPMENTS, FACT_DESIGN_WINS — that's the semi warehouse
3. Use that connector for all queries

Every query should:
- Apply `WHERE ORDER_TYPE <> 'Sample'` when aggregating FACT_ORDERS
- Join to DIM_DATE for FY filtering
- Return < 200 pre-aggregated rows (no raw selects on large facts)
