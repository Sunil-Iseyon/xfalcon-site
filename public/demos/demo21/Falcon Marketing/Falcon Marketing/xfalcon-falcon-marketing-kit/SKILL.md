# Falcon Marketing — Project Skill

This kit was generated by xFalcon AnalyticsPro on 2026-04-26 against a marketing agency warehouse. Everything below is locked-in for this project.

## Connector

`mcp__0f5a7fbd-d3a0-4d09-80d5-e325ec2e51bb__` — the only IDA connector visible. Tables are reachable without a `public.` prefix. Test with a simple `SELECT COUNT(*) FROM DIM_CLIENT` before any complex query.

## Schema (locked, do not re-discover)

**Facts:**
- `FACT_CAMPAIGN_PERFORMANCE` — 2.76M rows, grain: campaign × channel × geo × date
- `FACT_MEDIA_SPEND` — 54K rows, grain: campaign × channel × geo × month
- `FACT_PRODUCTION_JOBS` — 1.28M rows, grain: 1 row per JOB_CODE (verified — no multiplicity)
- `FACT_PROJECTS` — 980K rows, grain: 1 row per PROJECT_CODE (verified — service_line/employee never multiplied)

**Dimensions:** DIM_CLIENT (20), DIM_CAMPAIGN (1,035), DIM_CHANNEL (15), DIM_EMPLOYEE (120), DIM_SERVICE_LINE (15), DIM_GEOGRAPHY (40), DIM_JOB_TYPE (20), DIM_DATE (3,652), DIM_CREATIVE_ASSET (2,095, unjoinable to facts).

**Time:** 2018-01-01 → 2024-12-27 (campaigns) and 2017-01-01 → 2026-12-31 (DIM_DATE). Fiscal year = calendar year.

## Common column-name mistakes

| Wrong | Correct |
|-------|---------|
| `REVENUE` (campaign) | `REVENUE_ATTRIBUTED` |
| `COST` (production) | `TOTAL_COST` (per-job) or `UNIT_COST` |
| `BUDGET` (media) | `PLANNED_SPEND` |
| `IS_RUSH` | `RUSH_FLAG` |
| `IS_ONTIME` | `ON_TIME_FLAG` |
| `MARGIN` | `MARGIN_PCT` (decimal: 0.43 = 43%) |
| `EMPLOYEE_NAME` | doesn't exist — use `EMPLOYEE_CODE` and `ROLE_TITLE` |

## Required filters

**None hardcoded.** The user explicitly chose "no exclusions" — all clients, all statuses, all employees included. Do NOT add WHERE filters globally. Apply per-dashboard if a specific view requires it (e.g., active campaigns only on a current-state dashboard).

## Performance gotcha

Cross-fact joins (`FACT_PROJECTS` × `FACT_PRODUCTION_JOBS` × `DIM_CLIENT`) on the full 7-year range can timeout. **Always** aggregate each fact independently per (client, year), then combine in the dashboard or in a small UNION'd query.

## Pre-delivery QA — must-pass list

Run these on every HTML file before declaring "done":

1. CDN URL exact: `https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.js`
2. No `type: 'barh'` (use `type:'bar'` + `indexAxis:'y'`)
3. Every `<canvas id>` has a matching `new Chart()` or `mk()` call
4. All chart colors are HEX strings, never `var(--…)`
5. Back link `href="index.html"` (not `#`, not `javascript:history.back()`)
6. Topbar logo "xF", title = "Falcon Marketing / N. <Name>"
7. ≥6 KPI cards, ≥4 charts, ≥1 table with ≥7 rows + YoY column, ≥1 disclaimer
8. KPIs are single-year; multi-year requires Compare Years toggle
9. No red/green in chart datasets — only in KPI text
10. Theme inline script + theme.css link injected after `</title>`
11. theme.js loaded before `</body>`
12. Inter font from Google Fonts CDN
13. JS uses `var`, `function(){}`, string concatenation (no `const`/`let`/arrows/templates)
14. Numbers wrapped in `String()` when going into DOM helpers
15. `node --check` passes on the script block (no extra braces)
16. `fitKpiText()` defined and called in every `applyFilters()` and on load
17. Every filter `<select>` has `onchange="applyFilters()"`
18. Every `update*()` function reads from the `filtered` parameter, NOT global DATA
19. Reporting period subtitle reflects current filter
20. Filter dropdowns cleared (`innerHTML=''`) before populating in JS
